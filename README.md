## Usage

![Codespaces](https://img.shields.io/static/v1?style=for-the-badge&message=Codespaces&color=181717&logo=GitHub&logoColor=FFFFFF&label=)
![Devcontainers](https://img.shields.io/static/v1?style=for-the-badge&message=Devcontainers&color=2496ED&logo=Docker&logoColor=FFFFFF&label=)

⚠️ We only officially support [Debian](https://hub.docker.com/_/debian)-based
images. This includes all of the [official images] and the
[devcontainers-contrib images].

Most of the time, you can just add a feature to your [`devcontainer.json`] with
an empty object (`{}`) as the option value. Occasionally, you may need to
specify a version or other options. You can find these options on [our website].

```json
{
  "image": "mcr.microsoft.com/devcontainers/universal",
  "features": {
    "ghcr.io/devcontainers-contrib/features/deno": {},
    "ghcr.io/devcontainers-contrib/features/haskell": {}
  }
}
```

If you're looking to add a feature, we would love it if you did! ❤️ Please read
our [contributing guidelines] and the [how to create a feature] guide on [our
wiki]. If you just want to [suggest a feature], you can do that too!

This project is made using the [devcontainers CLI] to test, package, and publish
the feature container images. We also have a website built with [Eleventy].

Here are some useful commands:

| Command                         | Description                           |
| ------------------------------- | ------------------------------------- |
| `./configure`                   | Configure the project for development |
| `devcontainer features test`    | Test the features                     |
| `eleventy --input docs --serve` | Serve the website locally             |

💡 If you're using VS Code, you get these commands and more preset as [tasks].
